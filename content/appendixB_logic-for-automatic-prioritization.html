Appendix B: Logic for Automatic Prioritization of Review Checks

Explanation of the “Ranking Logic” Column in Table 6 below

(Refer to the Review Check Classifications section for full context and greater detail regarding meaning of ranking.)

Rank 0
  • Review check is automated via the Compliance Form and is always included.
  • Always assigned Rank 0.
Rank 1
  • Required for all Tiers 2–5.
  • Includes high-priority items like EUI thresholds and general PRM/ECB requirements (e.g., unmet load hours).
  • Always assigned Rank 1.
Rank 2
  • Always assigned Rank 2.
Rank 3
  • Always assigned Rank 3.
Rank 4
  • Always assigned Rank 4.
Rank 1 or 4
  • If the check affects an impactful end use, assign Rank 1.
  • Otherwise, assign Rank 4.
Rank 2, 3, or 4
  • Rank determined based on both the relevance of the check to impactful end uses and the project’s context:
    • Rank 2: Check targets an impactful end use and is especially significant due to building type, climate zone, or number of floors (e.g., a 24/7 facility in a cold climate).
    • Rank 3: Check targets an impactful end use, but without a context that makes it particularly critical.
    • Rank 4: Check does not target an impactful end use.
Rank 3 or 4
  • Rank 3: Check targets an impactful end use, but its relative impact is always lower.
  • Rank 4: Check does not target an impactful end use.
